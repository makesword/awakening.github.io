0x01.如何利用数据库的功能读写文件

读写条件： 1.要知道网站的绝对路径 2.要写的文件不能已经存在 3.当前数据库用户要有FILE权限 4.secure_file_priv的值为空

1.如何获取网站的绝对路径？ 通过访问phpinfo文件搜索document_root即可获得 通过报错来显示 2.如何查看用户的FILE权限

select user，file_priv from mysql.user;

3.secure-file-priv的作用

secure-file-priv参数是用来限制LOAD DATA, SELECT … OUTFILE, and LOAD_FILE()
传到哪个指定目录的。
当secure_file_priv的值为null ，表示限制mysqld 不允许导入|导出
当secure_file_priv的值为/tmp/ ，表示限制mysqld 的导入|导出只能发生在/tmp/
目录下
当secure_file_priv的值没有具体值时，表示不对mysqld 的导入|导出做限制
在mysql 5.6.34版本以后 secure_file_priv的值默认为NULL

4.secure_file_priv 值查看和修改 查看：

show global variables like 'secure_file_priv';

写文件操作：

select '<?php phpinfo(); ?>' into outfile 'F:/PHPstudy/phpstudy_pro/WWW/2.php';

读文件语句：

LOAD DATA LOCAL INFILE '文件' INTO TABLE 表;

0x02.数据库系统表学习

一般MySQL数据库初始的库有information_schema,mysql,performance_schema,sys四个 进行SQL注入时最常用的库是information_schema，这其中有四张特别重要的表，分别是schemata, tables, columns,user_privileges

schemata:当前mysql中所有数据库的信息 
tables:关于数据库中的表的信息 
columns:关于表中的列信息 
user_privileges:关于用户权限的信息
